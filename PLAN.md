# Clinical Feature Analysis Plan

This plan provides analysis tasks for the clinical kinematic
features derived from the pose estimation pipeline.  Each section
is self-contained and intended to be completed in a fresh coding
agent session.

**Prerequisites:** The `output/` directory already contains
`*_clinical.csv` (per-frame) and `*_clinical_windows.csv`
(per-window) files generated by `analysis/clinical_features.R`.
There are 12 videos (hands-arms mode), with row counts ranging
from 0 to 353 per-frame rows.

Mark a section `[x]` when complete.

---

## Section 1 — Explore and Sanity-Check Clinical Features

**Status:** [x] complete

**Goal:** Create `analysis/explore_clinical.R` that loads all
`*_clinical.csv` and `*_clinical_windows.csv` files from a
directory and produces an exploratory summary report with
visualisations, so the user can verify the features look
reasonable before further analysis.

**Context:**
- Per-frame features (18 numeric columns per row): joint angles
  (elbow, wrist deviation, finger spread), reach distance (raw
  and shoulder-normalised), grasp aperture (thumb–index,
  thumb–pinky), wrist displacement, and fingertip displacement
  — each for left and right sides.
- Per-window features (6 numeric columns): spectral arc length
  (SAL), mean wrist velocity, peak wrist velocity — each for
  left and right.
- Metadata columns: `video`, `frame_idx`, `timestamp_sec`,
  `person_idx` (per-frame); `video`, `person_idx`,
  `window_start_sec`, `window_end_sec` (per-window).
- Some videos are very short (6 rows) or empty (0 rows); the
  script should handle these gracefully.

**Steps:**
1. Load and combine all `*_clinical.csv` files (exclude files
   that also match `_clinical_windows`).  Similarly combine all
   `*_clinical_windows.csv` files.
2. Print a console summary: number of videos, total rows,
   per-video row counts, and per-column NA rates.
3. For each numeric feature, compute summary statistics (mean,
   median, SD, min, max, % NA) across all videos and print a
   tidy table.
4. Produce `*_distributions.png`: a multi-panel figure with one
   histogram (or density plot) per feature, coloured by video.
   Use `facet_wrap` with free scales.
5. Produce `*_na_heatmap.png`: a heatmap of missingness (rows =
   videos, columns = features, fill = proportion of NA frames).
6. Produce `*_boxplots.png`: box plots of each feature grouped
   by video, so between-video variation is visible at a glance.
7. For window features, produce `*_window_distributions.png`
   with the same multi-panel layout.
8. Print any data-quality warnings (e.g., features that are
   >50 % NA across all videos, videos with zero non-NA frames,
   constant-valued features).

**Deliverables:**
- `analysis/explore_clinical.R` — standalone Rscript.
  Usage: `Rscript analysis/explore_clinical.R output/`.
- PNG outputs written to the input directory with a shared
  prefix `all_clinical_`.
- Add a brief entry to `analysis/` usage in `README.md`.
- Update `AGENTS.md` with the new script and its outputs.

## Section 2 — Temporal Patterns Within Videos

**Status:** [x] complete

**Goal:** Create `analysis/temporal_clinical.R` that visualises
how clinical features evolve over time within each video,
revealing task structure and movement dynamics.

**Context:**
- Each `*_clinical.csv` has a `timestamp_sec` column giving the
  time of each frame.
- Features like reach distance and grasp aperture should show
  structured temporal patterns if the subject is performing
  reaching or grasping tasks.
- Window-level features (SAL, velocity) in
  `*_clinical_windows.csv` can be overlaid as step functions or
  ribbon annotations.

**Steps:**
1. For each video with ≥10 per-frame rows, produce a multi-panel
   time-series plot (`facet_wrap`, free y-scales) showing all
   per-frame features over `timestamp_sec`.  Plot left and right
   sides as separate coloured lines on the same panel where the
   feature name matches (e.g., `left_elbow_angle_deg` and
   `right_elbow_angle_deg` on one panel titled "Elbow Angle").
2. For videos with window-level data, add a second row of panels
   for SAL and velocity features (use `window_start_sec` as the
   x-axis).
3. Save one PNG per video:
   `<stem>_clinical_timeseries.png`.
4. Produce a single summary PNG (`all_clinical_timeseries_overview.png`)
   that shows one key feature (e.g., `right_reach_norm`) across
   all videos on a single faceted plot, for quick visual
   comparison of temporal profiles.

**Deliverables:**
- `analysis/temporal_clinical.R` — standalone Rscript.
  Usage: `Rscript analysis/temporal_clinical.R output/`.
- Per-video and summary PNGs written alongside the input CSVs.
- Update `README.md` and `AGENTS.md`.

## Section 3 — Between-Video Feature Comparison

**Status:** [x] complete

**Goal:** Create `analysis/compare_clinical.R` that aggregates
clinical features per video and compares them across videos to
identify subjects or sessions with notably different movement
patterns.

**Context:**
- Each video corresponds to one assessment session for one
  subject.
- Aggregating per-frame features to per-video summaries (mean,
  median, SD, min, max) reduces each video to a single row per
  feature, enabling direct comparison.
- Window features should be aggregated similarly (mean and SD of
  SAL and velocity across windows).

**Steps:**
1. Load all `*_clinical.csv` and `*_clinical_windows.csv` files.
2. Aggregate per-frame features per video: compute mean, median,
   SD, min, max of each numeric feature.  Aggregate window
   features per video: mean and SD of each.  Combine into one
   wide summary table.
3. Write `all_clinical_video_summary.csv` — one row per video,
   columns for each aggregated feature.
4. Produce `all_clinical_radar.png` (or parallel-coordinate
   plot): a multi-panel or overlay plot showing the profile of
   each video across key features (use z-scored means so
   features are on comparable scales).
5. Produce `all_clinical_heatmap.png`: a heatmap of z-scored
   feature means (rows = videos, columns = features), clustered
   by similarity to reveal groups.
6. Print a console summary flagging videos that are outliers
   (>2 SD from the group mean) on any feature.

**Deliverables:**
- `analysis/compare_clinical.R` — standalone Rscript.
  Usage: `Rscript analysis/compare_clinical.R output/`.
- CSV and PNG outputs in the input directory.
- Update `README.md` and `AGENTS.md`.

## Section 4 — Dimensionality Reduction on Clinical Features

**Status:** [x] complete

**Goal:** Create `analysis/clinical_dimreduce.R` that applies
PCA and UMAP to the per-video aggregated clinical features to
reveal clustering or structure among videos.

**Context:**
- The existing `analysis/features.R` does PCA and UMAP on raw
  landmark data.  This section applies the same idea to the
  higher-level clinical features.
- With only 12 videos, the primary value is visual: do videos
  group in a meaningful way, and which features drive the
  separation?
- The R package `uwot` is available for UMAP (already used by
  `analysis/features.R`).

**Steps:**
1. Load or recompute the per-video aggregated feature matrix
   (same as Section 3 step 2).  Drop columns with zero variance
   or >50 % NA.  Impute remaining NAs with column medians.
   Z-score all features.
2. Run PCA.  Produce:
   - `*_clinical_pca_scree.png` — scree plot of variance
     explained.
   - `*_clinical_pca_biplot.png` — PC1 vs PC2 biplot with
     video labels and feature loading arrows.
3. Run UMAP (n_neighbors = min(5, n-1) given the small sample).
   Produce:
   - `*_clinical_umap.png` — 2D UMAP with video labels.
4. Write `*_clinical_pca_loadings.csv` — tidy table of feature
   loadings on the first few PCs.
5. Print a console summary: top features loading on PC1 and PC2,
   total variance explained by first 2–3 PCs, any visible
   clustering interpretation.

**Deliverables:**
- `analysis/clinical_dimreduce.R` — standalone Rscript.
  Usage: `Rscript analysis/clinical_dimreduce.R output/`.
- Requires R packages `uwot` and `tidyverse`.
- PNG and CSV outputs in the input directory.
- Update `README.md` and `AGENTS.md`.

## Section 5 — Correlation and Longitudinal Readiness

**Status:** [x] complete

**Goal:** Create template metadata CSVs and a validation script
so the user can populate clinical scores and session metadata
for use with the existing `clinical_correlation.R` and
`longitudinal.R` scripts.

**Context:**
- `analysis/clinical_correlation.R` requires
  `clinical_scores.csv` with columns: `video` plus score columns.
- `analysis/longitudinal.R` requires `sessions.csv` with
  columns: `video`, `patient_id`, `session_date`.
- The user has 12 videos but may not yet have these metadata
  files populated.

**Steps:**
1. Create `analysis/make_templates.R` that:
   a. Scans the output directory for all unique `video` values
      across `*_clinical.csv` files.
   b. Writes `clinical_scores_template.csv` with one row per
      video and placeholder columns: `video`, `GRASSP`,
      `UEMS`, `SCIM` (values left blank for the user to fill).
   c. Writes `sessions_template.csv` with one row per video and
      columns: `video`, `patient_id`, `session_date` (values
      left blank).
   d. Prints instructions to the console explaining what to
      fill in and how to run the downstream scripts.
2. Create `analysis/validate_metadata.R` that:
   a. Reads a completed `clinical_scores.csv` or `sessions.csv`.
   b. Checks for: required columns present, no duplicate videos,
      video names match those in the clinical feature CSVs,
      dates parse correctly, scores are numeric.
   c. Reports errors and warnings clearly.
   d. Exits with status 0 if valid, 1 if errors found.

**Deliverables:**
- `analysis/make_templates.R` — standalone Rscript.
  Usage: `Rscript analysis/make_templates.R output/`.
- `analysis/validate_metadata.R` — standalone Rscript.
  Usage: `Rscript analysis/validate_metadata.R clinical_scores.csv output/`
  or `Rscript analysis/validate_metadata.R sessions.csv output/`.
- Template CSVs are written to the current directory (not
  committed — git-ignored like other output).
- Update `README.md` and `AGENTS.md`.
